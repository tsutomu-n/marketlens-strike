## 今回の目的

`review.md` を読んだ人間の判断を、機械が読める artifact として保存する。

出力先:

```text
data/strategy_reviews/{review_id}/operator_review.yaml
```

この artifact が表すもの:

```text
review packet を見た operator が、
「直す / 捨てる / 文脈として読んだ / paper bridge 検査候補にする」
のどれかを、対象 review packet の hash と一緒に記録したもの。
```

この artifact が表さないもの:

- paper order 作成許可
- paper intent 生成許可
- live 実行許可
- alpha 判定
- strategy case registry 登録
- NDX Layer 2.7 operator promotion 判定

## 決定した CLI

新規 command:

```bash
uv run sis strategy-review-record
```

判断を新規作成:

```bash
uv run sis strategy-review-record \
  --review-dir data/strategy_reviews/dogfood-complete-001 \
  --decision NEEDS_FIX \
  --reviewer tn \
  --rationale "lifecycle review is missing; regenerate review packet before candidate review" \
  --required-action "Generate lifecycle review and rebuild strategy review packet."
```

既存 `operator_review.yaml` の stale check:

```bash
uv run sis strategy-review-record \
  --review-dir data/strategy_reviews/dogfood-complete-001 \
  --validate-existing
```

CLI options:

| Option | Required | Meaning |
|---|---:|---|
| `--review-dir PATH` | yes | `review.md` と `review_manifest.json` がある directory |
| `--decision DECISION` | create mode only | operator の判断 |
| `--reviewer TEXT` | create mode only | reviewer id。空文字不可 |
| `--rationale TEXT` | create mode only | 判断理由。空文字不可 |
| `--required-action TEXT` | optional repeat | `NEEDS_FIX` の時は1件以上必須 |
| `--replace-existing` | no | 既存 `operator_review.yaml` を atomic replace する |
| `--validate-existing` | no | 既存 `operator_review.yaml` と現在の hash を照合する |

CLI exit:

| Case | Exit |
|---|---:|
| create success | 0 |
| validate existing success | 0 |
| missing `review_manifest.json` or `review.md` | 2 |
| invalid manifest JSON / Pydantic validation failure | 2 |
| invalid existing `operator_review.yaml` | 2 |
| stale existing `operator_review.yaml` | 2 |
| existing output without `--replace-existing` | 2 |
| invalid decision / invalid args | 2 |
| `PAPER_OBSERVATION_CANDIDATE` guard failure | 2 |

CLI stdout on create success:

```text
operator_review_path=data/strategy_reviews/{review_id}/operator_review.yaml
decision=NEEDS_FIX
review_id={review_id}
review_manifest_sha256=sha256:...
review_markdown_sha256=sha256:...
```

CLI stdout on validate success:

```text
operator_review_path=data/strategy_reviews/{review_id}/operator_review.yaml
status=valid
review_id={review_id}
```

## 決定した artifact contract

schema:

```text
schemas/operator_strategy_review.v1.schema.json
```

runtime model:

```text
src/sis/strategy_review/operator_review.py
```

YAML example:

```yaml
schema_version: operator_strategy_review.v1
review_id: dogfood-complete-001
reviewed_at: "2026-06-16T13:57:00Z"
reviewer: tn
decision: NEEDS_FIX
rationale: lifecycle review is missing; regenerate review packet before candidate review
required_actions:
  - Generate lifecycle review and rebuild strategy review packet.
live_allowed: false
paper_execution_allowed: false
source_review:
  manifest_path: data/strategy_reviews/dogfood-complete-001/review_manifest.json
  review_manifest_sha256: sha256:aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa
  review_markdown_path: data/strategy_reviews/dogfood-complete-001/review.md
  review_markdown_sha256: sha256:bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb
  review_status: INCOMPLETE_ARTIFACTS
  source_safety_status: UNKNOWN
  pack_validation_status: null
  lifecycle_review_status: missing
  missing_required_count: 0
  invalid_required_count: 0
  boundary_violation_count: 0
  unknown_boundary_count: 1
```

Required top-level fields:

- `schema_version`: const `operator_strategy_review.v1`
- `review_id`: same as `review_manifest.review_id`
- `reviewed_at`: timezone-aware UTC ISO string, `Z` suffix
- `reviewer`: non-empty string
- `decision`: enum
- `rationale`: non-empty string
- `required_actions`: list of non-empty strings, default `[]`
- `live_allowed`: const `false`
- `paper_execution_allowed`: const `false`
- `source_review`: object

Decision enum:

```text
REJECT
NEEDS_FIX
REVIEWED_FOR_CONTEXT
PAPER_OBSERVATION_CANDIDATE
```

Decision meaning:

| decision | Meaning | Can move forward |
|---|---|---|
| `NEEDS_FIX` | artifact 欠損、入力不備、説明不足、再生成が必要 | no |
| `REJECT` | 戦略または根拠が弱く、この review は打ち切る | no |
| `REVIEWED_FOR_CONTEXT` | 読んだが paper bridge 候補にしない。記録目的 | no |
| `PAPER_OBSERVATION_CANDIDATE` | paper bridge validation を試す候補として印を付ける | validation candidate only |

`source_review` required fields:

- `manifest_path`: repo-relative POSIX path
- `review_manifest_sha256`: `sha256:<64 lowercase hex>`
- `review_markdown_path`: repo-relative POSIX path
- `review_markdown_sha256`: `sha256:<64 lowercase hex>`
- `review_status`: `StrategyReviewManifest.review_status`
- `source_safety_status`: `StrategyReviewManifest.source_safety.status`
- `pack_validation_status`: string or null
- `lifecycle_review_status`: `present`, `missing`, `invalid`, or `blocked`
- `missing_required_count`: integer >= 0
- `invalid_required_count`: integer >= 0
- `boundary_violation_count`: integer >= 0
- `unknown_boundary_count`: integer >= 0

Model validation:

- `NEEDS_FIX` requires `required_actions` length >= 1.
- all decisions require non-empty `rationale`.
- `live_allowed` must be false.
- `paper_execution_allowed` must be false.
- sha256 fields must match existing `sha256:<64 lowercase hex>` format.
- paths must satisfy the same repo-relative POSIX safety rules as review manifest paths.
- extra fields are forbidden.
- `PAPER_OBSERVATION_CANDIDATE` is valid only when all of these are true:
  - `source_review.review_status == READY_FOR_HUMAN_REVIEW`
  - `source_review.source_safety_status == PASS`
  - `source_review.pack_validation_status == PASS`
  - `source_review.lifecycle_review_status == present`
  - `source_review.missing_required_count == 0`
  - `source_review.invalid_required_count == 0`
  - `source_review.boundary_violation_count == 0`
  - `source_review.unknown_boundary_count == 0`

Schema validation:

- JSON Schema must reject the same core invalid shapes as Pydantic.
- JSON Schema must reject `live_allowed=true`.
- JSON Schema must reject `paper_execution_allowed=true`.
- JSON Schema must reject bare hex hashes.
- JSON Schema must reject extra fields.
- JSON Schema must reject `NEEDS_FIX` without at least one `required_actions` item.
- JSON Schema must reject `PAPER_OBSERVATION_CANDIDATE` unless the fixed candidate conditions are visible in `source_review`.
