## 次の実作業

### PR-OPERATOR-00: Operator Strategy Review Artifact

目的:

`review.md` を読んだ人間の判断を、機械が読める artifact として保存する。

ここでは paper flow を呼びません。paper へ渡す前に、人間判断を構造化するだけです。

候補 CLI:

```bash
uv run sis strategy-review-record \
  --review-dir data/strategy_reviews/dogfood-complete-001 \
  --decision HOLD \
  --notes "paper intent preview が不足。authoring/lifecycle は確認済み。"
```

候補 artifact:

```yaml
schema_version: operator_strategy_review.v1
review_id: dogfood-complete-001
reviewed_at: 2026-06-16T22:57:00+09:00
decision: HOLD
notes: "paper intent preview が不足。authoring/lifecycle は確認済み。"
live_allowed: false
source_review:
  manifest_path: data/strategy_reviews/dogfood-complete-001/review_manifest.json
  review_manifest_sha256: sha256:...
```

推奨 decision enum:

```text
REJECT
REQUEST_REVISION
HOLD
PAPER_OBSERVATION_CANDIDATE
```

制約:

- `live_allowed=false` は schema / Pydantic で固定。
- `review_manifest.review_status` が `INVALID_INPUT` / `BLOCKED_BOUNDARY_VIOLATION` の場合、`PAPER_OBSERVATION_CANDIDATE` は不可。
- `review_manifest.source_safety.status != PASS` の場合、`PAPER_OBSERVATION_CANDIDATE` は不可。
- `review_id` 不一致は exit 2。
- `operator_review.yaml` は paper 実行許可ではなく、人間判断 artifact。
- 既存 `paper-from-intents` はまだ呼ばない。
- 既存 NDX Layer 2.7 の `ndx_operator_promotion_decision.v1` とは統合しない。まずは Strategy Review 専用 artifact として作る。
- YAML だけを初期 format にする。JSON 併用や DB 化はしない。
- `review_manifest_sha256` は生成時点の manifest hash。manifest が変わったら stale として扱う。

推奨対象ファイル:

```text
schemas/operator_strategy_review.v1.schema.json
src/sis/strategy_review/operator_review.py
src/sis/commands/strategy_review_operator.py
tests/strategy_review/test_operator_review_schema.py
tests/strategy_review/test_operator_review_cli.py
```

まだ不要:

```text
src/sis/strategy_review/review_decision.py
DB / registry
UI write API
paper bridge 呼び出し
```

完了条件:

```text
1. operator_review.yaml を生成できる
2. decision enum が固定されている
3. review_manifest path/hash が残る
4. INVALID / BLOCKED review に PAPER_OBSERVATION_CANDIDATE できない
5. live_allowed=false が固定される
6. stale review_manifest hash を検出して exit 2 できる
7. ./scripts/check が通る
```
