## 反映した前提

この計画は、既存 `marketlens-strike` repoでは `build-backtest` / `src/sis/backtest/bridge.py` が既存Strategy Lab / bridge系surfaceであり、新規のTrade[XYZ]純粋BT engineとは分ける、という前提で固定しました。

また、v0.1では **SP500 / long-only / single-symbol / market-like fill** に絞り、既存bridgeを触らず、CLIも後回しにして、まず `tests/backtest/*` で accounting / no-lookahead / cost / fill を固める方針にしています。

Trade[XYZ]側の外部仕様は、Hyperliquid / HIP-3上のXYZ marketsという前提、fee mode、funding、mark price、SP500仕様を公式Docsで再確認しました。Hyperliquid/HIP-3/XYZ protocolの位置づけはTrade[XYZ]公式Docsに明記されており、feeはstandard/growth mode、fundingはhourlyでoracle priceを使う式、mark priceはoracle・EWMA basis・bid/ask/last trade中央値のmedianとして説明されています。([Trade XYZ Docs][1])

## 今回の補正・改善点

特に事故りやすい点を修正しました。

```txt
1. 既存 build-backtest / bridge.py を新engineへ混ぜない
2. live側 TradeXyzOrderIntent をBT contractへ流用しない
3. fee hardcode禁止
4. funding_rateの単位推測禁止
5. unit testの runtime data/ 依存禁止
6. SPY -> SP500、QQQ -> XYZ100 の暗黙mapping禁止
7. MT5 / IC Markets / CFD は今回対象外
8. CLI公開はPR-8まで遅らせる
```

さらに、元資料の「gross pnl - fees - slippage - funding」型の表現は、**slippage二重控除リスク**があるため補正しました。v0.1では、cash accountingは `fill_price + fee + funding cashflow` で計算し、`slippage_impact` はreference priceとの差分としてreport/metricsに出します。fill priceにspread/slippageが既に織り込まれている場合、slippageをさらにPnLから引かない設計です。
