## 外部仕様との整合

Trade[XYZ]ではmark priceがmargining、liquidation、stop/limit trigger、unrealized P&Lに使われ、oracle price、oracle-mid差分の150秒EWMA、best bid/best ask/last trade中央値の3要素中央値として説明されています。したがって、BT側でも `last` や `close` だけで会計・リスク判定する設計は避けるべきです。([Trade XYZ Docs][1])

また、Discovery Boundsは内部価格形成中に特に重要で、mark priceを参照価格から一定範囲に制限します。v2では再アンカーがあり、単純な固定レンジではありません。MVPでは再計算せず、**entry gateとして「bound近辺では入らない」** までに留めるのが妥当です。([Trade XYZ Docs][2])

Trade[XYZ]のfeeはstandard / growth modeで変わるため、0.04%固定は使わず、Repo内の `configs/fee_model.trade_xyz.yaml` とrow側の `taker_fee_bps` / `maker_fee_bps` を優先する方針にしました。Trade[XYZ]公式もstandard feeとgrowth mode feeを分けて説明しています。([Trade XYZ Docs][3])

## 実装した主要contract

### TradeXyzMarketRow

現在のnormalized quote列を受けつつ、Trade[XYZ]固有の情報を保持します。

```text
ts
symbol
close
mid_price
mark_price
oracle_price
external_price
index_price
exec_buy_price
exec_sell_price
best_bid
best_ask
spread_bps
funding_rate
taker_fee_bps
maker_fee_bps
fee_mode
session_type
market_status
discovery_bound_bps
open_interest_usd
oi_cap_usd
oi_cap_usage
is_tradable
block_reasons
```

`ts_client` / `canonical_symbol` / `index_price` など既存normalized quote名からも読めるようにしています。

### fee model

優先順位はこれです。

```text
1. row.taker_fee_bps
2. row.fee_mode + configs/fee_model.trade_xyz.yaml
3. symbol classification + configs/fee_model.trade_xyz.yaml
4. 解決不能なら entry不可
```

`fee_mode=unknown` のまま勝手に0.04%へfallbackしない設計です。

### fill model

market-like fillの価格優先順位です。

```text
BUY:
  exec_buy_price
  best_ask
  ask_price
  mid_price + half spread
  mid_price
  close

SELL:
  exec_sell_price
  best_bid
  bid_price
  mid_price - half spread
  mid_price
  close
```

fill recordには `fill_price_source` を残します。
これは後で「どの価格仮定で勝ったのか」を監査するために必要です。

### portfolio accounting

long-onlyの最小会計です。

```text
entry:
  qty増加
  avg_price更新
  cash -= notional + fee
  realized_pnlは変えない

exit:
  qty減少
  realized_pnl += gross_pnl - fee
  cash += notional - fee
  flatならavg_price=0
```

### no-lookahead

シグナルはbar closeで生成し、**約定は次row**です。
同じrowの `exec_buy_price` をfillに使わないテストを追加しました。
