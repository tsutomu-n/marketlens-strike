Strategy Research Lab / Current Capabilities

今できることを、専門用語なしでも読める形で整理する

このページは、Strategy Research Lab が現在どこまで使えるかを具体的に説明するHTML版です。 ここで言う「使える」は、売買の作戦案を signal にし、paper-only の候補と仮注文案まで進められる、という意味です。

結論: 戦略研究から paper-only preview までは進められます。

まだ違う: live trading、実資金注文、収益性の証明ではありません。

Quick Status

一目で見る現在地

難しい言葉を抜くと、「売買アイデアを候補リストにして、紙上の仮注文案まで作れる」状態です。 ただし、勝てるかどうかの証明や実注文はまだ別工程です。

Can Do 売買シグナルの生成と記録
Can Do 複数候補の比較と paper 候補化
Not Yet 実資金の注文、PnL backtest、live-ready 判定
Flow

処理の流れ

各段階はファイルとして残ります。途中で止まった場合も、どこまで進んだか、なぜ止まったかを後から確認できます。

1. 作戦を選ぶ登録済み generator を指定する
2. signal を作る売買方向らしさを行データにする
3. trial に記録候補条件ごとの評価記録を残す
4. candidate 化paper に進める候補を束ねる
5. 人が判断hold / reject / promote を残す
6. 仮注文案paper-only preview を作る
7. 再検証最新 quote で paper runner が確認する
Capabilities

具体的にできること

左側は専門用語を避けた説明、右側は実際のコマンドや生成ファイルです。

できる

登録済みの作戦から売買シグナルを作る

あらかじめ登録された作戦を選び、売買方向・強さ・理由を持つデータを作れます。

コマンド: uv run sis strategy-preview

出力: strategy_signals.parquet, strategy_signal_manifest.json

補足: qqq_trend_rates_vix と sp500_trend_rates_vix を選べます。

できる

StrategyExperimentSpec から作戦を実行する

YAML/JSON の実験仕様を読み、登録済み generator を spec の lineage で実行できます。

コマンド: uv run sis strategy-experiment-run --spec path/to/strategy_experiment.yaml

出力: strategy_signals.parquet, strategy_signal_manifest.json

補足: parameter_grid は cartesian 展開され、variant ごとに parameter_hash が付きます。現行 built-in generator は min_source_confidence, max_vix_level / vix_gate, min_research_return_1d, timeframe を signal 条件または出力 timeframe として使えます。未登録 generator や上限超過は fail closed で止まり、live order は出しません。

できる

候補条件を変えて比較記録を作る

スコアがどの水準以上なら候補にするかを変えて、複数の試行記録を残せます。

コマンド: uv run sis evaluate-strategy-lab --rank-thresholds 0.2,0.8

出力: trial_ledger.jsonl

補足: これは比較記録であり、収益性の証明ではありません。

できる

複数のシグナルを paper 候補にする

最新の1件だけでなく、条件を通ったシグナルを複数まとめて候補化できます。

コマンド: uv run sis evaluate-strategy-lab --candidate-limit 0

次の操作: uv run sis build-paper-candidate-pack

出力: paper_candidate_pack.json

できる

時期ごとのシグナル数を記録する

日別・週別・月別などで、どのくらいシグナルが出たかを記録できます。

コマンド: --split-method walk_forward --era-unit trading_day

出力: metrics.era_signal_counts, metrics.era_count

補足: Strategy Lab評価CLIは signal count metrics、authoring backtest は era別 aggregate metrics を記録します。

できる

paper-only の仮注文案まで進める

人間の判断ファイルを経由して、paper runner に渡す仮注文案を作れます。

コマンド: promotion-decision, build-paper-intent-preview

出力: data/bot/paper_intent_preview.json

守る境界: paper_only=true, live_conversion_allowed=false

できる

YAML で戦略を書いて backtest する

entry / hold / close / reduce / add / rebalance / long / short / derived features including true range, ATR, Bollinger bands, Donchian channels, Keltner channels, Ichimoku cloud, MACD line, stochastic K/D, ADX, OBV, volume z-score, calendar features, rolling correlation / beta / spread z-score, flow/carry/liquidity/options-vol, on-chain/sentiment/event/fundamental/factor-ranking, execution-constraint, data-quality/ensemble/capacity features, lag, return/log-return, rolling return/sum/volatility, annualized volatility, realized variance, downside volatility, Sharpe/Sortino-like ratios, cumulative return, slope, mean-reversion score, EMA, RSI, and rolling min/max / column-to-column and cross/trend/consecutive condition / exclusion-none condition / regime membership filter / regime-specific overrides / paper-only dynamic multi-leg / pair-trade signal / group-wise cross-sectional top-bottom and fraction-tail rotation with minimum candidates and score thresholds / paper-only linear model score / train-model adapter / multi-timeframe confirmation panels / temporal-cadence control / event-window calendar filters / bracket-OCO lifecycle / opposite-signal exit / explicit close-signal exit / reduce-signal partial exit / add-signal scale-in / rebalance-signal exposure resize / order-style entry / slippage / partial-fill / spread gate / depth-based fill / latency gate / queue-position gate / short-borrow gate / stop-loss / take-profit / close-signal exit / partial exit / trailing stop / minimum holding period / sizing / grouped, group-net, and global net portfolio exposure limits / risk throttle / volatility targeting / target-weight / inverse-vol allocation / position-state pyramiding controls を YAML に書き、paper-only の fixed-horizon backtest と multi-strategy bundle / risk-parity allocation 比較まで実行できます。

Commands

よく使うコマンド

まずは default flow、その後に threshold sweep と複数候補化を使います。

最短で candidate pack まで

uv run sis strategy-preview
uv run sis evaluate-strategy-lab
uv run sis build-paper-candidate-pack

StrategyExperimentSpec から candidate pack まで

uv run sis strategy-experiment-run --spec path/to/strategy_experiment.yaml
uv run sis evaluate-strategy-lab
uv run sis build-paper-candidate-pack

SP500 generator を使う

uv run sis strategy-preview --generator-id sp500_trend_rates_vix
uv run sis evaluate-strategy-lab
uv run sis build-paper-candidate-pack

threshold sweep と複数候補

uv run sis evaluate-strategy-lab \
  --rank-thresholds 0.2,0.8 \
  --candidate-limit 0
uv run sis build-paper-candidate-pack

paper-only preview まで

uv run sis promotion-decision --decision hold
uv run sis build-paper-intent-preview
uv run sis paper-from-intents \
  --intents-path data/bot/paper_intent_preview.json

YAML 戦略を backtest する

uv run sis strategy-author-init \
  --out docs/strategy_research_lab/examples/trend_pullback_authoring_spec.yaml
uv run sis strategy-author-validate \
  --spec docs/strategy_research_lab/examples/trend_pullback_authoring_spec.yaml
uv run sis strategy-author-run \
  --spec docs/strategy_research_lab/examples/trend_pullback_authoring_spec.yaml \
  --through backtest
uv run sis strategy-author-bundle-run \
  --bundle docs/strategy_research_lab/examples/multi_strategy_authoring_bundle.yaml
Glossary

専門用語の言い換え

この repo 内の正確な名前と、普段の説明で使える言い換えを並べます。

用語 言い換え この repo での意味
generator シグナルを作る作戦部品 登録済みの戦略ロジック。例: qqq_trend_rates_vix。
signal 売買の気配、売買候補の元 方向、強さ、理由を持つ行データ。まだ注文ではない。
artifact 処理結果として残るファイル strategy_signals.parquet や trial_ledger.jsonl など。
TrialRecord 試した条件の記録 どの signal artifact を、どの条件で候補化したかを残す履歴。
rank threshold 候補にする最低点 rank_score がこの値以上の signal を候補として扱う。
candidate paper に進める売買候補 注文ではない。paper intent preview の材料。
paper-only 実資金を使わない紙上運用 wallet、署名、exchange write を使わない安全な観測モード。
PromotionDecision 人間の判断メモ candidate pack を hold / reject / promote する判断 artifact。
authoring YAML ユーザーが書く作戦ファイル strategy_authoring_spec.v1。entry、hold、long / short、true range、ATR、Bollinger bands、Donchian channels、Keltner channels、Ichimoku cloud、MACD line、stochastic K/D、ADX、OBV、volume z-score、calendar features、rolling correlation / beta / spread z-score、flow/carry/liquidity/options-vol, on-chain/sentiment/event/fundamental/factor-ranking, execution-constraint, data-quality/ensemble/capacity features、lag、EMA、RSI、rolling min/max/mean/std/z-score の派生列、損切、利確、部分利確、trailing stop、最低保有時間、sizing を paper backtest に渡す。
authoring bundle 複数作戦の比較束 strategy_authoring_bundle.v1。複数 YAML を allocation weight 付きで paper 比較する。
hold signal 今は入らない記録 side=none の研究 artifact。注文にも backtest trade にもしない。
PaperIntentPreview paper runner に渡す仮注文案 paper-only の意図。live order へ変換してはいけない。
walk-forward 時期を分けて検証する考え方 Strategy Lab評価CLIでは era count metrics、authoring backtestでは era別 aggregate metrics を記録する。
