1. これは何？

plain view

ざっくり言うと

現実の市場の動きと、Trade[XYZ] 側の価格を見比べて、「今は取引してよさそうか」を調べるための準備システムです。

6. Bot前の安全確認

before bot

価格は新しい？

古い価格で判断すると、地図を見ずに走るようなものです。

差が広すぎない？

買う値段と売る値段の差が大きいと、始めた瞬間に不利です。

量は足りる？

注文できる量が少ないと、思った値段で取引できません。

現実市場とズレすぎてない？

現実側とTrade[XYZ]側の価格が離れすぎていたら危険です。

情報源は信用できる？

ひとつの怪しいデータだけで決めないようにします。

危ない時間ではない？

市場が閉まっている時間や大きなニュースの近くは、待つ判断が必要です。

7. Bot化前に足りないもの

missing pieces

価格の健康診断

集めた価格が新しいか、ズレが大きすぎないか、注文できる量があるかを自動で見たいです。

Bot用の判断メモ

uv run sis bot-preview で「今は取引しない」と理由を書いたメモを出せます。

取引しない理由

Botは「買う」だけでなく、「今はやめる」と言える必要があります。その理由も人間に見える形にします。

注文の下書き

v1ではまだ注文候補を作りません。まず「待つ理由」を安全に出すところまでです。

一括更新コマンド

銘柄リスト、価格、現実市場、比較、判定をまとめて更新する入口があると、毎回の確認ミスが減ります。

11. よくある誤解

myth check

価格を集めた = Bot完成？

違います。価格は材料です。まだ安全判定と注文下書きが必要です。

仮想取引OK = 本番OK？

違います。仮想取引は練習です。本番には秘密鍵、wallet、失敗時の停止策が必要です。

micro liveがある = 自動売買OK？

違います。micro liveは小さな安全確認用です。普通のBotとして公開されていません。

12. 絶対に誤解しないこと

  • この段階では、本物のお金で自動売買しない。
  • 秘密鍵やwalletは使わない。
  • micro live は安全確認用のコードで、普通のBotとして公開されていない。
  • 古い gTrade / Ostium 用の判定を、Trade[XYZ] の準備完了と勘違いしない。
