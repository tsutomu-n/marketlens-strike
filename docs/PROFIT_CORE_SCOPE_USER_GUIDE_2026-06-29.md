# Profit Core User Guide

## 結論

この repo は、儲けを自動で見つける道具ではありません。

正しい使い方は、儲かった気になる前に止めることです。きれいなレポート、合格表示、画面、AI のコメントがあっても、それだけでは利益の証明ではありません。

## 本当に見るべきもの

見るべきことは多くありません。

1. 実際のお金に近い記録があるか。
2. 手数料、ズレ、待ち時間、失敗を入れても残るか。
3. 取引しない方がマシではないか。
4. 同じ日の同じような結果を何度も数えていないか。
5. 今日の判断が、続ける、止める、待つ、直すのどれかに落ちているか。

この 5 つに答えられないなら、まだ利益判断ではありません。

## NO_TRADE は失敗ではない

`NO_TRADE` は、取引しない判断です。

これは失敗ではありません。損を避けた判断です。

手数料やズレを入れると取引候補が薄くなるなら、何もしない方が良い場合があります。その時に `NO_TRADE` を無理に取引へ変えると、repo の意味がなくなります。

## 合格表示を信じすぎない

次の表示は、利益の証明ではありません。

- `PASS`
- `READY_FOR_HUMAN_REVIEW`
- `READ_ONLY_GO`
- paper fill
- viewer 完成
- dashboard 完成
- AI review
- たくさんの docs
- たくさんの CLI
- たくさんの artifact

これらは、確認や整理が進んだという意味です。儲かるという意味ではありません。

## 各機能の見方

| 名前 | どう見るか |
|---|---|
| Strategy Lab | 戦略候補を整理する場所。利益証明ではない |
| NDX | ローカル研究と検査の棚。alpha 証明ではない |
| Trade[XYZ] | 実装済み venue context。今の標準注文口ではない |
| Workbench | 結果を読みやすくする画面。判断そのものではない |
| AI Review | 人間が読むための材料。自動採用の許可ではない |
| audit / remediation | 壊れ方を見つけて直す作業。利益が出た証拠ではない |

便利な資料棚や検査道具はあります。ただし、それらは利益そのものではありません。

## 続ける前の質問

候補を続ける前に、次を自分に聞きます。

```text
実際のお金に近い証拠はあるか:
費用を入れても残るか:
悪い条件でも残るか:
NO_TRADE より良いか:
同じ結果を何度も数えていないか:
読むものが増えただけではないか:
今日の判断は keep / kill / wait / run のどれか:
```

答えが弱いなら、進めるより止める方が利益に近いです。

## 判断の意味

| 判断 | 意味 |
|---|---|
| keep | もう少し証拠を集める |
| kill | 捨てる。時間を使わない |
| wait | 日数、実測、入力、または市場状態を待つ |
| run | 条件を決めた範囲だけで試す |

`run` は、本番で大きく取引してよいという意味ではありません。金額、上限、口座、権限、止め方が決まっていないなら、まだ `run` ではありません。

## 迷った時の基準

迷ったら、こう読みます。

- 証拠が実際のお金から遠いなら、まだ待つ。
- 手数料やズレで消えるなら、殺す。
- `NO_TRADE` が一番良いなら、取引しない。
- 同じ日の再実行だけなら、強い証拠にしない。
- 画面やレポートが増えただけなら、進捗にしない。

この repo の一番大事な価値は、勝ちを保証することではありません。損しそうな候補、薄い候補、思い込みで良く見える候補を早く止めることです。
