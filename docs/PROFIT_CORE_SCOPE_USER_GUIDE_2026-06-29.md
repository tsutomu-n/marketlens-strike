# Profit Core User Guide

## 結論

この repo は、候補をたくさん作れます。ただし、候補が多いことは儲かった証拠ではありません。

正しい読み方は 3 段階です。

1. 候補を作る。
2. 弱い候補を backtest、費用、悪条件、`NO_TRADE` 比較で落とす。
3. 仮想約定で実行の形を見て、最後に実際のお金の記録だけを利益証拠として扱う。

Edge Candidate Factory の詳しい説明は [EDGE_CANDIDATE_FACTORY_CORE_2026-06-29.md](EDGE_CANDIDATE_FACTORY_CORE_2026-06-29.md) を読む。

## 候補生成は利益証明ではない

候補生成は、実データからたくさんの戦略候補を作る作業です。ここでは、良さそうな候補だけでなく、ダメだった候補、試した条件、落とした理由も残します。

これは大事な作業ですが、まだ利益ではありません。

- 候補が 1,000 個あっても、利益証明ではない。
- 市場イベントで 10,000 回発火しても、利益証明ではない。
- paper / demo / testnet で動いても、利益証明ではない。
- 実際の fill、fee、funding、cash ledger に接続して初めて、利益証拠として扱える。

## 4種類のサンプル

| 名前 | 意味 | どう読むか |
|---|---|---|
| candidate sample | 未検証候補の数 | 探索量。利益ではない |
| event sample | 候補が実データ上で発火した市場イベント数 | 検査材料。利益ではない |
| virtual forward sample | demo / testnet / paper account で見た仮想約定 | 実行の形を見る材料。利益ではない |
| actual cash sample | 実際のお金の損益記録 | 利益証拠として扱える最初の層 |

数字が大きいほど良いとは限りません。多くの候補を作る目的は、勝ちを保証することではなく、弱い候補を早く見つけて落とすことです。

## 本当に見るべきもの

見るべきことは多くありません。

1. 実際のお金に接続した記録があるか。
2. 手数料、ズレ、funding、待ち時間、失敗を入れても残るか。
3. 取引しない方がマシではないか。
4. 同じ日の同じような結果を何度も数えていないか。
5. AI や画面が許可したように見えていないか。
6. 今日の判断が、続ける、止める、待つ、条件付きで試すのどれかに落ちているか。

この質問に答えられないなら、まだ利益判断ではありません。

## NO_TRADE は失敗ではない

`NO_TRADE` は、取引しない判断です。

これは失敗ではありません。損を避けた判断です。

手数料やズレを入れると取引候補が薄くなるなら、何もしない方が良い場合があります。その時に `NO_TRADE` を無理に取引へ変えると、repo の意味がなくなります。

## LLM は許可する人ではない

LLM は、進める許可を出す人ではありません。止める理由を探す検査役です。

LLM に期待すること:

- 数字や説明の矛盾を探す。
- 抜けている証拠を探す。
- 言いすぎている主張を止める。
- 人間が読むための指摘を整理する。

LLM に任せないこと:

- 正式な PnL を決める。
- 正式 metric を決める。
- actual cash かどうかを判定する。
- live / tiny-live を許可する。
- 注文を作る。
- 戦略を書き換える。

`LLM_NO_ADDITIONAL_BLOCKER_FOUND` と出ても、許可ではありません。「追加で止める理由を見つけなかった」というだけです。

## 合格表示を信じすぎない

次の表示は、利益の証明ではありません。

- `PASS`
- `READY_FOR_HUMAN_REVIEW`
- `READY_FOR_HUMAN_RISK_REVIEW`
- `READ_ONLY_GO`
- `LLM_NO_ADDITIONAL_BLOCKER_FOUND`
- candidate sample の増加
- event sample の増加
- virtual forward sample の増加
- paper fill
- viewer 完成
- dashboard 完成
- AI review
- たくさんの docs
- たくさんの CLI
- たくさんの artifact

これらは、確認や整理が進んだという意味です。儲かるという意味ではありません。

## 各機能の見方

| 名前 | どう見るか |
|---|---|
| Edge Candidate Factory | 候補をたくさん作り、試したことと落とした理由を残す場所。利益証明ではない |
| Backtest Kill Gate | 弱い候補を落とす場所。利益証明ではない |
| Virtual Forward Execution Gate | 仮想約定で実行の形を見る場所。利益証明ではない |
| Actual Cash Report Gate | 実際のお金の記録とそれ以外を分ける場所 |
| Strategy Lab | 戦略候補を整理する場所。利益証明ではない |
| NDX / QQQ | v0 では単独の取引対象ではない。crypto perps の外部環境を見る材料として使える |
| Trade[XYZ] | 実装済み venue context。今の標準注文口ではない |
| Workbench | 結果を読みやすくする画面。判断そのものではない |
| AI Review | 止める理由を探す検査材料。自動採用の許可ではない |
| audit / remediation | 壊れ方を見つけて直す作業。利益が出た証拠ではない |

## 続ける前の質問

候補を続ける前に、次を自分に聞きます。

```text
これは候補数の話か、実損益の話か:
実際のお金に接続した証拠はあるか:
費用を入れても残るか:
悪い条件でも残るか:
NO_TRADE より良いか:
同じ結果を何度も数えていないか:
LLM が許可したように誤読していないか:
今日の判断は keep / kill / wait / run のどれか:
```

答えが弱いなら、進めるより止める方が利益に近いです。

## 判断の意味

| 判断 | 意味 |
|---|---|
| keep | もう少し証拠を集める |
| kill | 捨てる。時間を使わない |
| wait | 日数、実測、入力、または市場状態を待つ |
| run | 条件を決めた範囲だけで試す |

`run` は、本番で大きく取引してよいという意味ではありません。金額、上限、口座、権限、止め方が決まっていないなら、まだ `run` ではありません。

## 迷った時の基準

迷ったら、こう読みます。

- 候補生成なら、まだ利益ではない。
- 証拠が実際のお金から遠いなら、まだ待つ。
- 手数料やズレで消えるなら、殺す。
- `NO_TRADE` が一番良いなら、取引しない。
- 同じ日の再実行だけなら、強い証拠にしない。
- 画面やレポートが増えただけなら、進捗にしない。
- LLM が止めなかっただけなら、許可にしない。

この repo の一番大事な価値は、勝ちを保証することではありません。候補を広く作り、損しそうな候補、薄い候補、思い込みで良く見える候補を早く止めることです。
