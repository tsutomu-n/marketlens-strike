# 0212-Trend-Bot

## このノートの扱い

トレンドフォローbotの実戦寄りな骨格として使う。ただし完成戦略ではなく、レジーム判定、押し目/戻り、リスク管理、利確、停止条件の部品集として扱う。

## 元ノートの要旨

上位足でトレンドを判定し、短期足で押し目買い/戻り売りを行う。レバレッジ、マージンレシオ、ストップロス、段階的利確、トレーリング、時間フィルター、日次運用フローまで含む。

## 今日時点での補正

良い点は、エントリーよりも稼働/停止、証拠金、部分利確、急変時の対応に目が向いていること。補正すべき点は、トレンド判定が機能する市場だけを選ぶ条件と、レンジ相場での停止がまだ弱いこと。

## 理想的ナラティブ / 誤謬リスク

- `PREDICTION_OVERCLAIM`: 上位足トレンドがあるなら短期足の押し目は期待値がある、という前提。
- `EXECUTION_GAP`: 損切りや利確の価格に必ず近く約定できる、という前提。
- `BACKTEST_OVERFIT`: R倍利確やATRトレールのパラメータを後から最適化しやすい。

## 戦略部品への分解

- `Regime Detector`: 上位足トレンド、ADX、MA傾き、ボラティリティ。
- `Signal Generator`: 押し目/戻り、短期反転、出来高確認。
- `Position Sizer`: 1取引リスク、証拠金余力、連敗時縮小。
- `Exit Module`: 部分利確、ATRトレール、時間切れ決済。
- `Risk Guard`: 急変停止、日次損失上限、約定失敗停止。

## 実験に落とすなら

まず BTC/ETH の高流動性ペアだけで、上位足トレンドあり/なしを分けた単純ベースラインを作る。次に部分利確あり/なし、ATRトレールあり/なしを比較する。勝率ではなく、期待値、最大DD、連敗長、手数料込みPFを見る。

## 採用条件

レンジ期間を除外した時だけでなく、除外判定が遅れた期間でも損失が限定されること。

## 捨て条件

手数料とスリッページ込みで、単純なMAトレンドフォローを安定して上回らない場合。

## 現在性チェック

取引所の証拠金ルール、最小注文量、手数料体系、API制限は実装前に再確認する。
