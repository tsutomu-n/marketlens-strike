# 0725-時系列-予測モデル

## このノートの扱い

TimesFM、MOMENT、パッチング等の時系列モデルを、売買判断ではなく「予測研究の比較対象」として扱う。

## 元ノートの要旨

近年の時系列予測モデル、パッチング、基盤モデル的アプローチ、推論速度や表現力に関する内容。

## 今日時点での補正

時系列基盤モデルは魅力的だが、金融価格の短期予測では「予測精度が上がる」と「取引で勝てる」は別問題。価格そのものより、ボラティリティ、欠損補完、異常検知、レジーム特徴量として使う方が現実的。

## 理想的ナラティブ / 誤謬リスク

- `PREDICTION_OVERCLAIM`: 新しいモデルなら相場予測に強いという前提。
- `BACKTEST_OVERFIT`: モデル選定とfine-tuningで過去に合わせる。
- `DATA_VENDOR_DEPENDENCE`: 外部モデル、重い計算、ライセンス、再現性問題。

## 戦略部品への分解

- `Feature Factory`: forecast residual、vol forecast、trend probability。
- `Regime Detector`: 予測誤差拡大を相場変化として使う。
- `Research Assistant`: モデル比較と仮説生成。
- `Evaluation Harness`: naive forecast、ARIMA、EWMAとの比較。

## 実験に落とすなら

価格方向ではなく、翌期間ボラティリティ、レンジ幅、異常リスクを予測対象にする。naive baselineを必ず置く。

## 採用条件

単純なlag/rolling特徴量を上回り、取引判断に入れた時にコスト込み期待値が改善すること。

## 捨て条件

予測指標は良いが、売買に変換するとturnoverや手数料で消える場合。

## 現在性チェック

TimesFM、MOMENT等の公式リポジトリ、ライセンス、推奨環境、更新状況を確認する。
