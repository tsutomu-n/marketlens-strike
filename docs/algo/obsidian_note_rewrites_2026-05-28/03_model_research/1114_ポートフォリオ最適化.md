# 1114_ポートフォリオ最適化

## このノートの扱い

最適化で利益を増やす資料ではなく、複数戦略/複数銘柄を扱う時のリスク配分メモとして使う。

## 元ノートの要旨

ポートフォリオ最適化、リスク・リターン、分散、配分手法のメモ。

## 今日時点での補正

暗号資産や短期売買では、相関とボラティリティが非定常。平均分散最適化は入力推定誤差に弱いので、最初はequal weight、vol targeting、上限ルールから始めるべき。

## 理想的ナラティブ / 誤謬リスク

- `BACKTEST_OVERFIT`: 過去リターンと相関に最適化する。
- `PREDICTION_OVERCLAIM`: 将来の平均リターンを推定できる前提。
- `OPERATIONAL_COMPLEXITY`: 配分モデルが複雑化する。

## 戦略部品への分解

- `Portfolio Allocator`: equal weight、risk cap、vol target。
- `Position Sizer`: 銘柄別・戦略別の最大損失。
- `Risk Guard`: 相関急上昇、同時損失、流動性低下。
- `Evaluation Harness`: rebalance costとturnover評価。

## 実験に落とすなら

最適化手法より先に、単純配分と上限ルールの比較を行う。リバランス頻度別の手数料を含める。

## 採用条件

推定誤差に対して頑健で、銘柄追加時にDDが悪化しないこと。

## 捨て条件

最適化結果が少数銘柄に集中し、相場急変時に同時損失を出す場合。

## 現在性チェック

対象銘柄の流動性、手数料、先物/現物の制約を確認する。
