# 0507-戦略5 「トレンドの成熟度」判定によるイグジット戦略

## このノートの扱い

エントリー改善ではなく、利を伸ばす/降りる/縮小するためのExit Module候補として扱う。

## 元ノートの要旨

トレンドの成熟度を判定し、継続、部分利確、撤退を判断する。

## 今日時点での補正

成熟度は後から見れば分かるが、リアルタイムでは誤認しやすい。Exitは「天井を当てる」より「戻りをどこまで許容するか」を定義する方が実装しやすい。

## 理想的ナラティブ / 誤謬リスク

- `PREDICTION_OVERCLAIM`: トレンド終盤を判定できるという前提。
- `BACKTEST_OVERFIT`: 利確閾値や成熟度指標を過去に合わせやすい。
- `EXECUTION_GAP`: 急反転時に予定通り撤退できる前提。

## 戦略部品への分解

- `Exit Module`: ATRトレール、時間切れ、部分利確、勢い減衰。
- `Regime Detector`: トレンド強度低下、出来高変化、ボラ拡大。
- `Position Sizer`: 成熟度に応じた縮小。
- `Risk Guard`: 利益から損失への転落防止。

## 実験に落とすなら

固定Exit、ATR trail、成熟度Exitを比較する。勝率ではなく、平均利益、利益の取り逃し、最大含み益からの戻り、turnoverを見る。

## 採用条件

Exit追加で総利益だけでなく、利益の分布とDDが改善すること。

## 捨て条件

早降りが増えて、大きなトレンド利益を削る場合。

## 現在性チェック

実装時は取引所のreduce-only、OCO、trailing stop対応を確認する。
