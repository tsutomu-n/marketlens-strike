# 0507-戦略3 オンチェーンデータ強化型トレンド確信度モデル戦略

## このノートの扱い

オンチェーンデータを「価格予測の魔法」ではなく、参加可否や市場過熱度を補助する外部特徴量として扱う。

## 元ノートの要旨

SOPR、MVRV、アクティブアドレス、取引所流入出、資金調達率などでトレンド継続の確信度を作る。

## 今日時点での補正

オンチェーン指標は銘柄、チェーン、データベンダーに依存する。日次指標は短期売買には遅く、使うなら中期レジームや過熱警戒が中心。

## 理想的ナラティブ / 誤謬リスク

- `DATA_VENDOR_DEPENDENCE`: Glassnode/CryptoQuant/Dune等の定義差を軽視する。
- `PREDICTION_OVERCLAIM`: オンチェーンをファンダメンタルズとして過信する。
- `BACKTEST_OVERFIT`: 指標選択と閾値が後知恵になりやすい。

## 戦略部品への分解

- `Data Collector`: 指標定義、更新頻度、遅延、欠損管理。
- `Feature Factory`: z-score、変化率、過熱/冷却フラグ。
- `Regime Detector`: risk-on/off、蓄積/分配、過熱。
- `Participation Filter`: トレンドシグナルの採用/縮小。

## 実験に落とすなら

オンチェーン指標を入れない価格ベースラインと比較する。追加指標は一度に1つずつ。日次更新なら日足以上に限定する。

## 採用条件

データ取得時点で利用可能だった値だけを使い、未使用期間でもDD削減か見送り改善が出ること。

## 捨て条件

指標の更新遅延や改訂を考慮すると優位性が消える場合。

## 現在性チェック

各オンチェーンデータベンダーの指標定義、料金、API制限、履歴修正を確認する。
