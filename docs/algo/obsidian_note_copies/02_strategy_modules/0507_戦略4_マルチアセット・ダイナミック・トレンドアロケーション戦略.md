**戦略4: マルチアセット・ダイナミック・トレンドアロケーション戦略**

**コンセプト:**
複数の仮想通貨に対して個別にトレンドフォロー戦略を適用し、各通貨のトレンドの強さ、ボラティリティ、および通貨間の相関を考慮して、ポートフォリオ全体の資金配分を動的に調整する戦略。リスク分散を図りつつ、市場全体のトレンドや有望な個別トレンドを捉えることを目指す。

**OSSからの着想点:**

*   **複数アセットへの戦略適用:** `commodity_trend_following_chinese_futures.ipynb` では、多数の中国のコモディティ先物に対してトレンドフォロー戦略を適用し、個々のシャープレシオを分析しています。これは、戦略を単一アセットだけでなく、ユニバース全体に広げる発想の基礎となります。
*   **リスク調整と分散:** `cross_asset_skewness.ipynb` は、異なるアセットクラス（コモディティ、株式、債券、通貨）に分散してスキューネス戦略を適用する例です。同様に、仮想通貨内でも異なる特性を持つ通貨に分散することでリスクを管理できます。`realised_volatility.ipynb` でのボラティリティ・スケーリングは、個々のアセットのリスク貢献度を均等化する上で重要です。
*   **パフォーマンス統計の活用:** `futures_long_only.ipynb` では、多数の先物契約のパフォーマンス統計（CAGR、ボラティリティ、シャープレシオなど）を集計しています。このような統計的アプローチは、アロケーション判断の基礎となり得ます。

**詳細ロジック:**

**対象資産 (ユニバース):**

1.  **選定基準:**
    *   時価総額: 例としてトップ20〜50位。
    *   流動性: 主要取引所での十分な取引量と狭いスプレッド。
    *   データの可用性: 信頼できる価格データと（可能であれば）オンチェーンデータが取得可能。
    *   プロジェクトの質: ホワイトペーパー、開発チーム、コミュニティ活動なども考慮（定性的要素）。
2.  **カテゴリ分け (オプション):**
    *   レイヤー1 (BTC, ETH, SOL, ADAなど)
    *   レイヤー2 (MATIC, ARB, OPなど)
    *   DeFi (UNI, AAVE, LDOなど)
    *   ミームコイン (DOGE, SHIBなど、高リスクだがトレンドが出やすい場合も)
    *   このカテゴリ分けは、後のアロケーションでセクター集中リスクを管理するのに役立つ。

**時間軸:**
日足データを基本とし、週次または月次でポートフォリオのアロケーションを見直す。個別のトレンドシグナルは日次で発生。

**ステップ1: 個別資産トレンド戦略の実行**

ユニバース内の各仮想通貨に対して、標準化されたトレンドフォロー戦略を適用する。

1.  **トレンドシグナル生成:**
    *   戦略1（適応型ボラティリティ・スケーリング付きマルチシグナル・トレンド戦略）の簡易版、またはよりシンプルなEMAクロスオーバー（例：20日EMA vs 50日EMA）など。
    *   各通貨のシグナル（ロング=+1, ショート=-1, ニュートラル=0）を毎日生成。

2.  **ボラティリティ・スケーリング (仮):**
    *   この段階では、各通貨がポートフォリオに組み入れられた場合の「仮の」目標ボラティリティ（例：年率60%など、やや高めに設定）に基づいて、仮のポジションサイズを計算しておく。これは、後でポートフォリオ全体のリスクを調整するための入力となる。
    *   `仮ポジションサイズ = 仮目標ボラティリティ / 個別実現ボラティリティ`

**ステップ2: トレンド強度と魅力度のスコアリング**

各通貨のトレンドの強さや魅力を定量的にスコアリングする。

1.  **トレンド強度スコア:**
    *   ADXの値。
    *   トレンド持続期間（現在のトレンドシグナルが何日間継続しているか）。
    *   直近N日間のトレンドフォロー戦略のパフォーマンス（シミュレーションベース）。
    *   複数の指標を組み合わせ、正規化してスコア化（例: 0〜1）。

2.  **ボラティリティ・リスクスコア (低いほど良い):**
    *   直近の実現ボラティリティの絶対水準。
    *   ボラティリティのボラティリティ（安定性）。
    *   これらを組み合わせて、スコアが高いほどリスクが高い（魅力度が低い）と評価。

3.  **総合魅力度スコア:**
    *   `総合魅力度スコア = w_trend * トレンド強度スコア - w_risk * ボラティリティ・リスクスコア`
    *   重み `w_trend`, `w_risk` はバックテストで最適化。

**ステップ3: 通貨間相関の分析**

1.  **相関行列の計算:**
    *   ユニバース内の各通貨ペアの日次リターンの過去N日間（例：60日〜120日）の相関行列を計算。

**ステップ4: ポートフォリオ構築とアロケーション**

総合魅力度スコアと相関行列を考慮して、各通貨への資金配分比率を決定する。

1.  **アロケーション手法の選択:**
    *   **ランクベース・均等加重（修正版）:**
        1.  総合魅力度スコアで通貨をランキング。
        2.  上位N通貨（トレンドシグナルがオンのもの）を選定。
        3.  選定された通貨に均等に資金を割り当てるが、相関が高い通貨群がある場合は、その群全体のウェイトに上限を設けるか、群内の通貨ウェイトを調整する。
    *   **階層的リスクパリティ (Hierarchical Risk Parity - HRP):**
        1.  通貨間の相関構造に基づいて階層的クラスタリングを行う。
        2.  クラスタ内で逆ボラティリティ加重を行い、クラスタ間でさらに逆ボラティリティ加重または均等加重を行うことで、リスクを分散させる。総合魅力度スコアをウェイトの調整因子として組み込むことも可能。
    *   **平均分散最適化 (Mean-Variance Optimization - MVO) の制約付き応用:**
        1.  総合魅力度スコアを期待リターンの代理変数とし、相関行列と個別ボラティリティをリスクとして、シャープレシオ最大化などの目的関数で最適化。
        2.  ただし、期待リターンの推定誤差が大きいため、ウェイトの上限/下限設定、セクター配分制約などの強い制約条件を課す必要がある。

2.  **最終ポジションサイズ決定:**
    *   各通貨へのアロケーション比率が決まったら、ポートフォリオ全体の目標ボラティリティ（例：年率30%）を達成するように、全体のレバレッジを調整し、各通貨の最終的なポジションサイズを決定する。
    *   `最終ポジションサイズ_i = アロケーション比率_i * ポートフォリオ資金 * (ポートフォリオ目標Vol / 予想ポートフォリオVol) / 資産価格_i`

**ステップ5: リバランス**

*   **定期的リバランス:** 週次または月次で、ステップ2〜4を再実行し、ポートフォリオのウェイトを目標ウェイトに調整する。
*   **トリガー式リバランス (オプション):** ポートフォリオのウェイトが目標から一定以上乖離した場合や、主要通貨のトレンドシグナルが大きく変化した場合にも臨時リバランスを検討。

**パラメータの例 (初期設定):**

*   ユニバース: 時価総額トップ30仮想通貨
*   個別トレンドシグナル: 20日EMA vs 50日EMA
*   トレンド強度スコア: ADX(14) + トレンド持続日数 (正規化)
*   相関計算期間: 90日
*   アロケーション手法: ランクベース（上位10通貨、相関考慮）
*   ポートフォリオ目標ボラティリティ: 年率30%
*   リバランス頻度: 週次

**バックテストと最適化:**

*   多数の通貨データを扱うため、データ管理と処理効率が重要。
*   アロケーション手法の選択と、そのパラメータ（HRPのクラスタリング手法、MVOの制約条件など）の最適化。
*   各通貨の個別トレンド戦略のパラメータも影響するため、感度分析が必要。
*   取引コスト（特にリバランス時）を考慮したネットパフォーマンスの評価。
*   異なる市場環境でのポートフォリオの安定性、最大ドローダウンの検証。

**留意点:**

*   **流動性リスク:** アルトコインの中には、大きなポジションを取ると価格に影響を与えたり、スリッページが大きくなったりするものがある。アロケーションウェイトの上限設定などで対応。
*   **相関の不安定性:** 仮想通貨間の相関は時間とともに大きく変化する可能性があり、過去の相関が将来も続くとは限らない。
*   **集中リスク:** 特定のセクター（例：DeFi関連銘柄）にトレンドが集中した場合、ポートフォリオがそのセクターに偏るリスクがある。カテゴリ分けやセクターウェイト上限の設定で対応。
*   **計算複雑性:** 特にHRPやMVOを用いる場合、計算負荷が高くなる。

この戦略は、個別のトレンドフォロー戦略をポートフォリオレベルで統合し、リスク管理とリターン向上を目指すものです。OSSの `commodity_trend_following_chinese_futures.ipynb` のような個別アセットの分析から一歩進んで、ポートフォリオ全体としての最適化を試みるアプローチです。成功すれば、より洗練されたリスク・リターン特性を持つトレンドフォロー戦略となる可能性があります。

def run_multi_asset_trend_strategy(price_data_dict_full, daily_returns_dict_full, params):
    """ リバランス日にアロケーションを決定し、目標ポジション（ドル建て）を返す """
    all_dates = price_data_dict_full[params['asset_universe'][0]].index # どれか一つの日付インデックスを取得
    rebalance_dates = pd.date_range(start=all_dates.min(), end=all_dates.max(), freq=params['rebalance_frequency'])
    
    portfolio_allocations_history = {}

    for rb_date in rebalance_dates:
        if rb_date not in all_dates: continue # データがない日はスキップ

        # リバランス日までの過去データを使用
        current_price_data_dict = {
            asset: df[df.index <= rb_date] for asset, df in price_data_dict_full.items()
        }
        current_daily_returns_dict = {
             asset: series[series.index <= rb_date] for asset, series in daily_returns_dict_full.items()
        }
        
        # 1. 個別アセットのシグナルとメトリクス計算
        asset_metrics = get_individual_asset_signals_and_metrics(current_price_data_dict, params)
        
        # 2. ポートフォリオアロケーション計算
        target_weights_for_date = calculate_portfolio_allocation(asset_metrics, current_daily_returns_dict, params)
        
        # 3. ドル建ての目標ポジションに変換
        target_positions_value = {
            asset: params['portfolio_capital'] * weight
            for asset, weight in target_weights_for_date.items()
        }
        portfolio_allocations_history[rb_date] = target_positions_value
        
    return pd.DataFrame.from_dict(portfolio_allocations_history, orient='index').fillna(0)
