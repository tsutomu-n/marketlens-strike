**戦略1: 適応型ボラティリティ・スケーリング付きマルチシグナル・トレンド戦略**

**コンセプト:**
短期・中期・長期の複数のトレンドシグナルを組み合わせることでシグナルの信頼性を高め、長期的な市場環境をフィルタリングしつつ、日々の実現ボラティリティに基づいてポジションサイズを動的に調整することで、リスクをコントロールしながらトレンドを捉える戦略。

**OSSからの着想点:**

*   **マルチシグナル:** `trend_following_baltas2020.ipynb` では、異なる期間の移動平均や価格モメンタムを組み合わせることで、よりロバストなトレンド判断を目指すアプローチが見られます。
*   **長期トレンドフィルター:** `equity_short_term_trading_connors.ipynb` では、200日移動平均線を長期トレンドの判定に利用し、その方向に従った取引のみを行う例があります。
*   **ボラティリティ・スケーリング:** `realised_volatility.ipynb` で様々な実現ボラティリティ推定法が比較されており、特にYang-Zhang推定器などが売買回転率を抑えつつ安定したボラティリティ目標を達成するのに有効であることが示唆されています。`trend_following_baltas2020.ipynb` でもボラティリティ・スケーリングは重要な要素として扱われています。
*   **レバレッジ上限:** `realised_volatility.ipynb` の実験で `signal_cap=0.95` が使われていたように、過度なレバレッジを避けるためのキャップ設定は実務上重要です。

**詳細ロジック:**

**対象資産:**
流動性の高い主要な仮想通貨（例: BTC, ETH, SOLなど）。各通貨ペア（例: BTC/USD, ETH/USD）に対して個別に戦略を適用。

**時間軸:**
日足データを基本としてシグナル生成とポジション調整を行う。

**ステップ1: 長期トレンドフィルター**

1.  **指標:** 200日指数平滑移動平均線 (EMA) を計算。
2.  **判定:**
    *   当日の終値 > 200日EMA → 長期上昇トレンド
    *   当日の終値 < 200日EMA → 長期下降トレンド
    *   当日の終値 ≒ 200日EMA → 長期トレンド不明確（オプション：この場合は取引を見送るか、ポジションサイズを縮小）

**ステップ2: マルチトレンドシグナル生成**

以下の3つのトレンドシグナルを計算し、統合する。

1.  **シグナルA (短期EMAクロスオーバー):**
    *   **指標:** 12日EMAと26日EMA (伝統的なMACDの期間設定に近い)。
    *   **シグナル:**
        *   12日EMA > 26日EMA → 短期上昇シグナル (+1)
        *   12日EMA < 26日EMA → 短期下降シグナル (-1)
        *   それ以外 → ニュートラル (0)

2.  **シグナルB (中期チャネルブレイクアウト):**
    *   **指標:** 過去55日間の高値（アッパーチャネル）と安値（ロワーチャネル）。 (Donchian Channel)
    *   **シグナル:**
        *   終値がアッパーチャネルをブレイク → 中期上昇シグナル (+1)
        *   終値がロワーチャネルをブレイク → 中期下降シグナル (-1)
        *   それ以外 → ニュートラル (0)
        *   （オプション：一度ブレイクしたら、反対側のチャネルをブレイクするまでシグナルを継続）

3.  **シグナルC (ADXによるトレンド強度判定):**
    *   **指標:** 14日間ADX (Average Directional Index) と +DI, -DI。
    *   **シグナル:**
        *   ADX > 25 かつ +DI > -DI → トレンド強度あり・上昇方向 (+1)
        *   ADX > 25 かつ +DI < -DI → トレンド強度あり・下降方向 (-1)
        *   ADX <= 25 → トレンド強度なし・レンジ相場 (0)

**ステップ3: シグナルの統合と最終トレンド方向の決定**

1.  **加重:** 各シグナルに重み付けを行う（初期は均等ウェイトで良いが、将来的には過去のパフォーマンスに基づいて最適化）。
    例: `最終スコア = (wA * シグナルA) + (wB * シグナルB) + (wC * シグナルC)`
2.  **最終トレンド方向:**
    *   最終スコア > 閾値 (例: 0.5) → 最終上昇トレンド
    *   最終スコア < -閾値 (例: -0.5) → 最終下降トレンド
    *   それ以外 → 最終トレンド不明確

**ステップ4: ポジションサイジング (適応型ボラティリティ・スケーリング)**

1.  **実現ボラティリティ (RV) の計算:**
    *   `realised_volatility.ipynb` で検討されたOHLCベースの推定器、特にYang-Zhang推定器またはGarman-Klass Yang-Zhang推定器を日次で計算。計算期間は過去20〜60日程度。
    *   これを年率換算する (`RV_annualized`)。

2.  **目標ボラティリティ (TargetVol):**
    *   ポートフォリオ全体、または各資産に対して年率の目標ボラティリティを設定（例: 40%）。これはリスク許容度に応じて決定。

3.  **基本ポジションサイズ (PositionSize_raw):**
    *   `PositionSize_raw = TargetVol / RV_annualized`

4.  **長期トレンドフィルターによる調整:**
    *   ステップ1の長期トレンドフィルターが「上昇トレンド」で、ステップ3の最終トレンド方向も「上昇トレンド」の場合: `PositionSize_adjusted = PositionSize_raw` (ロングポジション)
    *   ステップ1の長期トレンドフィルターが「下降トレンド」で、ステップ3の最終トレンド方向も「下降トレンド」の場合: `PositionSize_adjusted = PositionSize_raw` (ショートポジション)
    *   ステップ1とステップ3の方向が一致しない場合、またはどちらかが「不明確」な場合: `PositionSize_adjusted = PositionSize_raw * 0.5` (ポジションサイズを半減)、または取引を見送る(0)。

5.  **レバレッジ上限 (LeverageCap):**
    *   `PositionSize_final = min(PositionSize_adjusted, MaxLeverage)`
    *   `MaxLeverage` は固定値（例: 2.0）または過去の計算ポジションサイズの95パーセンタイル値などで設定。

**ステップ5: エントリーとイグジット**

1.  **エントリー:**
    *   翌日の始値で、ステップ4で計算された `PositionSize_final` に基づいてポジションを建てる。
    *   ロングの場合: `PositionSize_final > 0`
    *   ショートの場合: `PositionSize_final < 0` (ただし、ショートが可能な市場/商品であること)

2.  **イグジット (損切り/利益確定/シグナル反転):**
    *   **損切り:** ATR (Average True Range) に基づくトレーリングストップ。例えば、エントリー価格から `N * ATR` (Nは2〜3程度) 離れた位置に初期ストップを設定し、価格が有利に動けばストップラインを引き上げる。
    *   **利益確定:** 固定のRRR (Risk-Reward Ratio、例: 1:3) に基づくターゲット、またはトレンド転換シグナル（ステップ3の最終トレンド方向が反転）発生時。
    *   **シグナル反転:** ステップ3の最終トレンド方向が現在のポジションと反対方向に転換した場合、現在のポジションを決済し、新しい方向にドテンする。

**ステップ6: 定期リバランス**

*   毎日、終値ベースでステップ1〜4を再計算し、翌日の目標ポジションサイズを決定する。
*   現在のポジションサイズと目標ポジションサイズに乖離がある場合、翌日の始値で差分を調整する（リバランス）。
*   ただし、売買回転率を抑えるため、一定以上の乖離（例：目標サイズの10%以上）がなければリバランスしないなどの閾値を設けることも検討。

**パラメータの例 (初期設定):**

*   長期EMA: 200日
*   短期EMAクロス: 12日EMA, 26日EMA
*   中期チャネルブレイクアウト: 55日
*   ADX: 14日, ADX閾値25
*   シグナル加重: `wA=0.4, wB=0.3, wC=0.3` (仮)
*   最終スコア閾値: 0.5
*   実現ボラティリティ計算期間: 21日 (Yang-Zhang)
*   目標ボラティリティ: 年率40%
*   最大レバレッジ: 2.0倍
*   ATR期間 (損切り用): 14日, ATR倍率: 2.5

**バックテストと最適化:**

*   上記ロジックとパラメータをOSSの `vivace` ライブラリの `BacktestEngine` のようなバックテストフレームワークで実装。
*   過去データ（少なくとも数年分、異なる市場環境を含む）で徹底的にバックテストを行う。
*   シャープレシオ、最大ドローダウン、勝率、平均損益率、売買回転率などを評価。
*   ウォークフォワード分析やモンテカルロシミュレーションなどでパラメータのロバスト性を確認。
*   各種パラメータ（EMA期間、チャネル期間、ADX設定、目標ボラティリティ、ATR設定など）を最適化。

**留意点:**

*   仮想通貨市場は24時間365日動いているため、日足ベースの戦略でも、どの時点の価格（例：UTC 0時の終値）を基準にするか明確にする。
*   取引所のAPIを利用した自動売買を想定。APIのレート制限、接続安定性、注文執行の遅延なども考慮。
*   取引手数料とスリッページはパフォーマンスに大きな影響を与えるため、現実的に見積もる。

このロジックは、OSSで示された複数の有効なコンセプトを組み合わせ、仮想通貨市場の特性（高ボラティリティ、トレンド発生の可能性）に対応しようとするものです。各ステップの具体的な指標やパラメータは、さらなる研究と検証によって調整されるべきです。

import pandas as pd
import numpy as np
# 実運用ではTA-Libのようなライブラリの利用を推奨
# import talib

# --- データ準備 (仮定) ---
# price_df: pd.DataFrame (index=DatetimeIndex, columns=['Open', 'High', 'Low', 'Close', 'Volume'])
# 各仮想通貨に対してこのロジックを適用

# --- ユーティリティ関数 ---
def calculate_ema(series, period):
    """ 指数平滑移動平均 """
    return series.ewm(span=period, adjust=False).mean()

def calculate_donchian_channel(high_series, low_series, period):
    """ ドンチアンチャネル """
    upper = high_series.rolling(window=period).max()
    lower = low_series.rolling(window=period).min()
    return upper, lower

def calculate_atr(high_series, low_series, close_series, period):
    """ Average True Range (ATR) - Wilder's Smoothing """
    tr1 = high_series - low_series
    tr2 = abs(high_series - close_series.shift(1))
    tr3 = abs(low_series - close_series.shift(1))
    tr = pd.concat([tr1, tr2, tr3], axis=1).max(axis=1, skipna=False)
    atr = tr.ewm(alpha=1/period, adjust=False).mean()
    return atr

def calculate_adx(high_series, low_series, close_series, period):
    """ ADX, +DI, -DI (TA-Lib推奨だが、概念実装) """
    df = pd.DataFrame({'High': high_series, 'Low': low_series, 'Close': close_series})
    df['UpMove'] = df['High'].diff()
    df['DownMove'] = df['Low'].diff().mul(-1) # Lowの差分を正にする

    df['+DM_raw'] = np.where((df['UpMove'] > df['DownMove']) & (df['UpMove'] > 0), df['UpMove'], 0)
    df['-DM_raw'] = np.where((df['DownMove'] > df['UpMove']) & (df['DownMove'] > 0), df['DownMove'], 0)

    # Wilder's Smoothing for DM and TR
    atr_for_adx = calculate_atr(df['High'], df['Low'], df['Close'], period)
    
    df['+DM_smooth'] = df['+DM_raw'].ewm(alpha=1/period, adjust=False).mean()
    df['-DM_smooth'] = df['-DM_raw'].ewm(alpha=1/period, adjust=False).mean()

    df['+DI'] = 100 * (df['+DM_smooth'] / atr_for_adx)
    df['-DI'] = 100 * (df['-DM_smooth'] / atr_for_adx)
    
    # DXとADXの計算
    # +DIと-DIの合計が0になるケースを避けるため、小さな値を加算
    di_sum = df['+DI'] + df['-DI']
    di_sum = np.where(di_sum == 0, 1e-6, di_sum) # ゼロ除算回避
    
    df['DX'] = 100 * (np.abs(df['+DI'] - df['-DI']) / di_sum)
    adx = df['DX'].ewm(alpha=1/period, adjust=False).mean()
    return adx, df['+DI'], df['-DI']

def calculate_gkyz_volatility(df_ohlc, window=21, trading_periods_per_year=365):
    """ Garman-Klass-Yang-Zhang (GKYZ) ボラティリティ推定 (概念的実装)
        正確な実装は論文参照や専門ライブラリを推奨。
        ここでは主要なコンポーネントの組み合わせイメージを示す。
    """
    # 夜間ボラティリティ (前日終値 -> 当日始値)
    log_return_overnight = np.log(df_ohlc['Open'] / df_ohlc['Close'].shift(1))
    var_overnight = log_return_overnight.rolling(window=window).var()

    # 日中ボラティリティの要素 (Garman-Klass like)
    # u = ln(High/Open), d = ln(Low/Open), c = ln(Close/Open)
    u = np.log(df_ohlc['High'] / df_ohlc['Open'])
    d = np.log(df_ohlc['Low'] / df_ohlc['Open'])
    c_intra = np.log(df_ohlc['Close'] / df_ohlc['Open'])
    
    # Garman-Klass estimator (日中部分の分散の一例)
    # 0.5 * (ln(H/L))^2 - (2ln2 - 1) * (ln(C/O))^2
    # ここでは簡単のため、Garman-Klassの主要項を使用
    var_intraday_gk_like = (0.5 * np.log(df_ohlc['High'] / df_ohlc['Low'])**2 - \
                           (2 * np.log(2) - 1) * np.log(df_ohlc['Close'] / df_ohlc['Open'])**2)
    var_intraday_gk_like = var_intraday_gk_like.rolling(window=window).mean()


    # Yang-Zhangの調整係数k (簡略化、原論文参照)
    k_yz = 0.34 / (1.34 + (window + 1) / (window - 1))

    # GKYZはこれらの要素を組み合わせるが、ここでは簡略的な加算で代替
    # 実際には、Yang-Zhang (1999) や GKYZ (Garman, Klass, Yang, Zhang) の論文の正確な式を用いること
    # total_variance_estimate = var_overnight + k_yz * var_intraday_gk_like + (1-k_yz) * (log_return_overnight**2).rolling(window=window).mean()
    # 上記はYang-Zhangの一要素であり、GKYZはさらに調整が入る
    # より入手しやすいRogers-Satchellボラティリティも代替として考えられる
    # 簡略化のため、ここではローリング標準偏差を使用するが、OHLCベースが望ましい
    log_return_close = np.log(df_ohlc['Close'] / df_ohlc['Close'].shift(1))
    simple_vol = log_return_close.rolling(window=window).std() * np.sqrt(trading_periods_per_year)
    
    # 最小ボラティリティの設定 (例: 過去1年の10パーセンタイル)
    min_vol_threshold = simple_vol.rolling(window=trading_periods_per_year).quantile(0.10)
    
    return np.maximum(simple_vol, min_vol_threshold.fillna(0.01)) # 極端に低いボラを避ける

# --- メインロジック関数 ---
def generate_trading_signals_and_positions(price_df, params_dict):
    df = price_df.copy()

    # --- ステップ1: 長期トレンドフィルター ---
    df['long_term_ema'] = calculate_ema(df['Close'], params_dict['long_term_ema_period'])
    df['long_term_trend'] = np.sign(df['Close'] - df['long_term_ema']) # 1, -1, 0

    # --- ステップ2: マルチトレンドシグナル生成 ---
    # シグナルA (EMAクロス + 乖離フィルター)
    df['ema_short_A'] = calculate_ema(df['Close'], params_dict['ema_short_A_period'])
    df['ema_mid_A'] = calculate_ema(df['Close'], params_dict['ema_mid_A_period'])
    ema_spread_A = (df['ema_short_A'] - df['ema_mid_A']) / df['ema_mid_A'] # 乖離率
    df['signal_A'] = 0
    df.loc[(df['ema_short_A'] > df['ema_mid_A']) & (ema_spread_A > 0.005), 'signal_A'] = 1 # 0.5%以上乖離
    df.loc[(df['ema_short_A'] < df['ema_mid_A']) & (ema_spread_A < -0.005), 'signal_A'] = -1

    # シグナルB (ドンチアンチャネル + 出来高フィルター)
    df['donchian_upper_B'], df['donchian_lower_B'] = calculate_donchian_channel(df['High'], df['Low'], params_dict['donchian_B_period'])
    df['avg_volume_B'] = df['Volume'].rolling(window=params_dict['donchian_B_period']).mean()
    df['signal_B'] = 0
    # 前日のチャネルと出来高で判断
    df.loc[(df['Close'] > df['donchian_upper_B'].shift(1)) & (df['Volume'] > df['avg_volume_B'].shift(1) * params_dict['donchian_B_volume_factor']), 'signal_B'] = 1
    df.loc[(df['Close'] < df['donchian_lower_B'].shift(1)) & (df['Volume'] > df['avg_volume_B'].shift(1) * params_dict['donchian_B_volume_factor']), 'signal_B'] = -1
    # シグナル継続ロジック (ATR Stopや反対シグナルで抜けるまで)
    # df['signal_B'] = df['signal_B'].replace(0, method='ffill').fillna(0) # 注: このffillは慎重に検討

    df['adx_C'], df['plus_di_C'], df['minus_di_C'] = calculate_adx(df['High'], df['Low'], df['Close'], params_dict['adx_C_period'])
    df['signal_C'] = 0
    df.loc[(df['adx_C'] > params_dict['adx_C_trend_threshold']) & (df['plus_di_C'] > df['minus_di_C']), 'signal_C'] = 1
    df.loc[(df['adx_C'] > params_dict['adx_C_trend_threshold']) & (df['plus_di_C'] < df['minus_di_C']), 'signal_C'] = -1
    # ADX低下によるトレンド終了警告 (イグジットで利用)
    df['adx_exit_warning'] = df['adx_C'] < params_dict['adx_C_exit_threshold']


    # --- ステップ3: シグナルの統合 (加重平均とスコアの連続化) ---
    # 各シグナルの強さを -1 から +1 で表現 (例: EMA乖離率をtanhでスケール)
    # strength_A = np.tanh(ema_spread_A * 20) # スケーリング係数は要調整
    # strength_B = df['signal_B'] # ドンチアンは1/0/-1のまま
    # strength_C = np.select([df['signal_C'] == 1, df['signal_C'] == -1], [df['adx_C']/50, -df['adx_C']/50], default=0).clip(-1,1) # ADX値を強さに

    # ここでは単純な多数決＋長期フィルターで代替
    df['combined_signal_temp'] = df['signal_A'] + df['signal_B'] + df['signal_C']
    df['final_trend_direction'] = 0
    df.loc[(df['combined_signal_temp'] >= 2) & (df['long_term_trend'] == 1), 'final_trend_direction'] = 1  # 3つのうち2つ以上がロング、かつ長期もロング
    df.loc[(df['combined_signal_temp'] <= -2) & (df['long_term_trend'] == -1), 'final_trend_direction'] = -1 # 3つのうち2つ以上がショート、かつ長期もショート


    # --- ステップ4: ポジションサイジング ---
    df['realized_vol'] = calculate_gkyz_volatility(df[['Open','High','Low','Close']], window=params_dict['volatility_window'])
    # ゼロやNaNを小さな値で埋めてエラー回避、ただし実運用では要注意
    df['realized_vol_safe'] = df['realized_vol'].replace(0, np.nan).fillna(method='ffill').fillna(0.01) 
    
    # 目標エクスポージャー (例: 1単位の価格変動がポートフォリオの TargetVol/sqrt(365) になるように)
    # target_daily_risk_exposure = price_df['Close'] * (params_dict['target_annual_vol'] / np.sqrt(365))
    # num_contracts_raw = target_daily_risk_exposure / (df['realized_vol_safe'] / np.sqrt(365) * price_df['Close'])
    # 上記は複雑なので、元のポジションサイズ計算に戻す
    df['position_size_raw'] = params_dict['target_annual_vol'] / df['realized_vol_safe']

    # 長期トレンドフィルターと最終シグナル方向の一致でフルサイズ、それ以外は0
    df['position_size_adjusted'] = df['position_size_raw']
    df.loc[df['final_trend_direction'] == 0, 'position_size_adjusted'] = 0 # トレンドなしは0

    # 最終的な目標ポジション（レバレッジ考慮）
    # ポジションサイズはレバレッジと解釈（例：1なら1倍レバレッジ）
    df['target_leverage'] = (df['position_size_adjusted'] * df['final_trend_direction']).fillna(0)
    df['target_leverage'] = np.clip(df['target_leverage'], -params_dict['max_leverage'], params_dict['max_leverage'])


    # --- ステップ5: イグジットシグナル (ATRトレーリングストップの計算準備) ---
    df['atr_exit'] = calculate_atr(df['High'], df['Low'], df['Close'], params_dict['atr_period_exit'])
    
    # 実際のポジション管理とリバランスは別モジュールで行う想定
    # この関数は日々の目標レバレッジとATRを返す
    return df[['final_trend_direction', 'target_leverage', 'atr_exit', 'adx_exit_warning']]
