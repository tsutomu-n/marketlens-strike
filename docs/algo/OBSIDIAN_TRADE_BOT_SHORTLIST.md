## まず掴める気づき

- 戦略の軸は1本ではなく、少なくとも `トレンド追随` `板/マイクロ構造` `予測モデル` `配分最適化` `執行最適化` `ニュース/イベント駆動` の6系統が見える。
- vault 内では、単一の聖杯ロジックよりも「複数フィルタを重ねて誤発火を減らす」思想が多い。
- 価格シグナル単体より、`長期トレンド判定 -> 短期エントリー -> リスク制御 -> 執行補助` の分業で考える方が筋が良い。
- `Order Book` と `Trend` 系ノートが両方あるので、順張り戦略でも「方向は長期足、タイミングは板」で分離できる。
- `LightGBM` や `時系列予測` は、いきなり売買判断を全自動化するよりも、`トレード可否フィルタ` や `レジーム判定` に使う方が現実的。
- `ポートフォリオ最適化` は複数銘柄配分だけでなく、単一戦略の `サイズ配分` `レバレッジ配分` `戦略間 capital allocation` に転用できる。
- `遺伝的アルゴリズム` は万能ではないが、手作業で思いつかないパラメータ結合やルール結合を探す用途には相性が良い。
- `PyBotters` と取引所/チェーン系ノートがあるので、研究で終わらず実行レイヤまで落とし込む土台はある。
- `0507-戦略1..5` は、単発メモではなく戦略設計の型としてまとまっており、発想源としての密度が高い。

## 戦略アイデアの種

- `長期トレンド + 短期押し目 + 板確認`
  - 例: `0212-Trend-Bot.md` で方向判定し、`0714-トレード戦略-Order-Book.md` でエントリー可否を絞る。
- `トレンド戦略 + 異常検知フィルタ`
  - 例: `0714-Adaptive-Alpha-Trendトレードプログラム.md` の異常検知を、荒れ相場回避フィルタとして使う。
- `予測モデルは売買判断ではなく regime 分類に使う`
  - 例: `0721-LightGBM パラメータチューニング.md` と `1101_時系列予測.md` を使い、今が trend/range/high-vol のどれかを分類する。
- `板情報は方向予測より execution 改善に使う`
  - 例: スリッページが重い局面だけ見送る、板の偏りが消えたら利確する、など。
- `ニュース収集は signal ではなく watchlist 生成に使う`
  - 例: `1107_自動化された暗号通貨ニュースで稼ぐ方法.md` や `0702-cryptofetch.md` を、当日監視銘柄やイベント検知に使う。
- `配分最適化を strategy-of-strategies に使う`
  - 例: トレンド追随、逆張り、イベント駆動の3系統に capital を配る。
- `トレンドの成熟度を exit 専用ロジックにする`
  - 例: `0507-戦略5_「トレンドの成熟度」判定によるイグジット戦略.md` を、既存エントリー戦略の exit module として流用する。
- `オンチェーン確信度を participation filter にする`
  - 例: `0507-戦略3_オンチェーンデータ強化型トレンド確信度モデル戦略.md` を、Solana 系の参加可否判定に使う。
- `遺伝的アルゴリズムを entry 条件ではなく threshold 探索に使う`
  - 例: ADX、ATR 倍率、板不均衡閾値、時間停止閾値の探索。

## 最初の仮説として良さそうな組み合わせ

- 仮説1: `Trend-Bot + Order Book + PyBotters`
  - 狙い: トレンド相場だけ参加し、板で無駄撃ちを減らす。
- 仮説2: `Adaptive Alpha Trend + LightGBM + VectorBT`
  - 狙い: 指標戦略に機械学習フィルタを重ね、検証を高速反復する。
- 仮説3: `Polars + VectorBT + 遺伝的アルゴリズム`
  - 狙い: 特徴量と閾値の組み合わせ探索を高速化する。
- 仮説4: `ニュース収集 + regime 判定 + 実行 Bot`
  - 狙い: 普段は待機し、イベント性のある日だけ戦略を強める。
- 仮説5: `Trend Entry + Maturity Exit`
  - 狙い: 入り方はシンプルに保ち、降り方だけ賢くする。
- 仮説6: `Multi-Asset Trend + Dynamic Allocation`
  - 狙い: 銘柄選択と資金配分を一体で設計する。

## 読むときの観点

- このノートは `方向判定` `タイミング` `サイズ` `損切り` `利確` `停止条件` `執行` `研究速度` のどこに効くか。
- 単体で強そうかではなく、別ノートと組み合わせた時に役割分担がきれいか。
- シグナルの精度向上より、誤発火を減らすか、損失の肥大化を抑えるか、検証回転を速めるかで評価する。
