**戦略5: 「トレンドの成熟度」判定によるイグジット戦略**

**コンセプト:**
標準的なトレンドフォローシグナルでエントリーした後、トレンドが継続している間は利益を伸ばしつつ、トレンドの勢いが衰えたり、終焉の兆候が見られたりした場合に、固定の損切りラインや利益目標に達する前に早期に手仕舞うことで、利益を確保し、大きなドローダウンを回避する戦略。

**OSSからの着想点:**

*   **トレンドの非永続性:** 多くのトレンドフォロー戦略のバックテスト結果（例：`trend_following_moskowitz2012.ipynb` の長期パフォーマンス）は、トレンドが永遠には続かず、いずれ終了または反転することを示しています。利益確定と損切りは不可欠ですが、より賢いイグジット方法があればパフォーマンスを改善できる可能性があります。
*   **市場パターンの変化:** `equity_etf_intraday_momentum.ipynb` でのOLS係数の時間変化は、市場のパターンや予測力が一定ではないことを示唆します。トレンドも同様に、初期、中期、終期で異なる特徴を示す可能性があります。
*   **ボラティリティと出来高の重要性:** `realised_volatility.ipynb` はボラティリティの動的な変化を捉えることの重要性を示しており、`commodity_crack_spread_stat_arb.ipynb` の価格チャート分析では、価格変動と出来高の関係性が（暗に）示されています。これらはトレンドの勢いを測る上で重要な指標です。
*   **オシレーターの活用:** 伝統的なテクニカル分析では、RSIやMACDのようなオシレーター系指標のダイバージェンスがトレンド転換の先行指標として利用されることがあります。

**詳細ロジック:**

**対象資産:**
流動性の高い主要な仮想通貨。

**時間軸:**
日足データを基本とし、トレンド成熟度の判定も日次で行う。

**ステップ1: エントリー**

1.  **トレンドフォローシグナル:** 戦略1（適応型ボラティリティ・スケーリング付きマルチシグナル・トレンド戦略）や戦略2（レジームスイッチング戦略）のような、堅牢なトレンドエントリーシグナルを用いる。
    *   例: 50日EMAが200日EMAを上抜いた場合にロングエントリー。
2.  **初期損切り設定:** エントリーと同時に、ATR (Average True Range) に基づく初期損切りラインを設定。
    *   例: ロングエントリーの場合、エントリー価格 - `2.5 * ATR(14)`。

**ステップ2: トレンド成熟度・枯渇度モニタリング指標**

ポジションを保有している間、以下の指標を毎日監視する。各指標は、トレンドの勢いが弱まっている、または反転の可能性が高まっていることを示唆する方向に動いた場合に「警告ポイント」を加算するイメージ。

1.  **ボラティリティの変化:**
    *   **指標a (ボラティリティの急低下):** 短期実現ボラティリティ（例：過去10日間のYang-Zhangボラティリティ）が、中期実現ボラティリティ（例：過去50日間）と比較して大幅に低下（例：50%以下になるなど）。トレンド終盤の「凪」を示唆。
    *   **指標b (ボラティリティの急上昇と逆行):** ポジション保有方向と反対の方向に大きな価格変動があり、かつボラティリティが急上昇した場合（例：ロング中に大陰線が出てDVOLが急騰）。パニック的なトレンド終焉の可能性。
2.  **出来高の変化:**
    *   **指標c (出来高の減少):** トレンド方向に価格が進んでいるにも関わらず、出来高が過去N日間（例：20日）の移動平均を大きく下回り続ける。トレンド継続のエネルギー不足を示唆。
3.  **オシレーターのダイバージェンス:**
    *   **指標d (RSIダイバージェンス):**
        *   ロングポジション中: 価格は高値を更新しているが、RSI(14)が高値を更新しない（弱気のダイバージェンス）。
        *   ショートポジション中: 価格は安値を更新しているが、RSI(14)が安値を更新しない（強気のダイバージェンス）。
    *   **指標e (MACDダイバージェンス):** MACDラインやヒストグラムで同様のダイバージェンスを確認。
4.  **価格アクションとサポート/レジスタンス:**
    *   **指標f (主要移動平均線への回帰と反落/反発失敗):**
        *   ロングポジション中: 価格が短期EMA（例：20日EMA）まで調整し、そこから再上昇できずにEMAを明確に下抜ける。
        *   ショートポジション中: 同様に、短期EMAまで戻し、そこから再下落できずにEMAを明確に上抜ける。
    *   **指標g (重要なサポート/レジスタンスラインでの停滞):** 過去に何度も意識された価格水準でトレンドが停滞し、出来高を伴って反転の動きを見せる。
5.  **トレンド持続期間と利益水準:**
    *   **指標h (期間ベースの警告):** トレンドが異常に長く（例：過去の平均トレンド持続期間の2倍以上）継続している場合、統計的に反転確率が高まっていると見なす。
    *   **指標i (利益水準ベースの警告):** ポジションが非常に大きな含み益（例：初期リスクの5倍以上）を抱えている場合、一部利益確定を検討するシグナル。

**ステップ3: 早期イグジット判定ロジック**

1.  **警告ポイントシステム:**
    *   ステップ2の各指標（a〜i）が警告条件を満たした場合、事前に設定されたポイントを加算。
        *   例: RSIダイバージェンス発生 → +2ポイント、出来高減少 → +1ポイントなど。
        *   ポイントの大きさは、各指標の過去の予測力に基づいて決定。
2.  **イグジット閾値:**
    *   合計警告ポイントが事前に設定した閾値（例: 5ポイント）を超えた場合、早期イグジットシグナルとする。
3.  **段階的イグジット (オプション):**
    *   閾値を複数設定し、低い閾値に達したらポジションの一部を決済、高い閾値で全決済するなどの段階的アプローチも可能。

**ステップ4: イグジット実行**

1.  早期イグジットシグナルが発生した場合、翌日の始値でポジションの全部または一部を決済する。
2.  これは、ステップ1で設定したATRトレーリングストップや固定の利益目標に達する「前」に行われる。
3.  もちろん、ATRトレーリングストップに先にヒットした場合は、その時点で損切りが行われる。

**パラメータの例 (初期設定):**

*   エントリー: 50日EMA vs 200日EMAクロス
*   初期損切り: エントリー価格 - 2.5 * ATR(14)
*   **モニタリング指標と警告ポイント (仮):**
    *   ボラティリティ急低下 (10日RV < 50日RVの50%): +1pt
    *   出来高減少 (20日平均の70%以下が3日継続): +1pt
    *   RSI(14)ダイバージェンス (2回連続で確認): +2pt
    *   MACD(12,26,9)ダイバージェンス (ヒストグラムで確認): +1pt
    *   20日EMA反落/反発失敗 (終値で明確にブレイク): +2pt
*   早期イグジット閾値 (合計警告ポイント): 5ポイント

**バックテストと最適化:**

*   各モニタリング指標のパラメータ（期間設定、閾値など）と、それらに割り当てる警告ポイントの最適化が鍵。
*   早期イグジット閾値の感度分析。
*   「早期イグジットしなかった場合」のパフォーマンスと比較し、戦略の有効性を評価（例：利益は減るがドローダウンが大幅に改善されるか）。
*   異なる市場環境（高ボラティリティ期、低ボラティリティ期）でのロバスト性を確認。

**留意点:**

*   **指標の組み合わせの複雑性:** 多くの指標を組み合わせると、過学習のリスクが高まる。シンプルさを保ちつつ、効果的な指標を見つけるバランスが重要。
*   **「ダマシ」のシグナル:** 各成熟度指標も完璧ではなく、トレンドがまだ継続するにも関わらず早期イグジットしてしまう「ダマシ」のシグナルを出す可能性がある。
*   **主観性の排除:** 警告ポイントの割り当てや閾値設定において、客観的な基準（バックテスト結果など）に基づいて決定する努力が必要。

この戦略は、トレンドフォロー戦略の永遠の課題である「いつ手仕舞うか」に対して、より動的で状況適応的なアプローチを取ろうとするものです。OSSでは直接的なイグジット戦略の高度化はあまり見られませんが、トレンドの非永続性や市場パターンの変化といった普遍的な観察から着想を得ています。成功すれば、プロフィットファクターの向上やドローダウンの抑制に貢献する可能性があります。

**この改善版スニペットのポイント:**

1.  **モニタリング指標の具体化とパラメータ化:**
    *   RSIダイバージェンスについて、「強い」「弱い」の概念と、それを判定するための具体的な価格とRSIの動きの条件、および高値圏/安値圏の閾値（`rsi_divergence_threshold_strong`など）をパラメータ化しました。
    *   MACDについても、ラインクロスとヒストグラムダイバージェンスを個別の警告ポイントとして考慮できるようにしました。
    *   ボラティリティの変化について、ATRを用いた収縮と（不利な方向への）急拡大を検知するロジックを追加しました。
2.  **警告ポイントシステムの柔軟性:**
    *   各警告シグナルに対して割り当てるポイント（`points_rsi_strong_divergence`など）をパラメータ化し、バックテストによる調整を容易にしました。
3.  **段階的イグジットの導入:**
    *   `exit_warning_threshold_partial` と `exit_warning_threshold_full` という2段階の閾値を設定し、警告の度合いに応じて一部利益確定または全決済を行うロジックの枠組みを示しました。`partial_exit_ratio` で一部決済の割合を指定できます。
4.  **関数の責務分離:** `check_trend_maturity_and_generate_exit_signals` 関数は、あくまで現在の市場状況とポジションから警告ポイントとイグジット「シグナル」を返すことに集中し、実際のポジション変更や損切りヒット判定は呼び出し元のポジション管理ループで行う想定としました。
5.  **履歴データの利用:** ダイバージェンスや移動平均の計算には一定期間の履歴データが必要なため、`price_history_df_for_exit` を関数に渡す形にしました。
6.  **コメントの充実:** 各指標の意図や、パラメータの調整方針、さらなる改善点などをコメントで補足しました。

**依然として残る課題・簡略化されている点:**

*   **ダイバージェンス判定の厳密性:** RSIやMACDのダイバージェンス判定は、ピーク/ボトムを正確に特定するアルゴリズム（例：`scipy.signal.find_peaks`など）と組み合わせることで、より堅牢になります。現在のスニペットは簡易的な比較です。
*   **警告ポイントと閾値の最適化:** これらは依然としてバックテストと統計的分析によって客観的に決定されるべきです。
*   **利益保護ストップの具体ロジック:** 「ブレークイーブン・ストップ」や「利益額に応じたトレーリングストップの積極化」などの具体的な計算ロジックはまだ詳細化されていません。
*   **指標間の複合条件:** 「出来高減少し、かつRSIダイバージェンス発生」のような複合的な条件でのポイント加算はまだ考慮されていません。

このスニペットは、トレンドの終焉をより多角的に捉え、柔軟なイグジット判断を行うための具体的な設計案を示しています。各指標のパラメータや警告ポイントの重み付けを最適化することで、より効果的なイグジット戦略を構築できる可能性があります。
