# 0507-戦略1 適応型ボラティリティ・スケーリング付きマルチシグナル・トレンド戦略

## このノートの扱い

マルチシグナル型トレンド戦略の候補。主役はエントリーではなく、ボラティリティに応じたサイズ調整と見送り条件。

## 元ノートの要旨

複数のトレンド/モメンタム指標を組み合わせ、ボラティリティに応じてポジションサイズを調整する。

## 今日時点での補正

複数シグナルは堅牢化ではなく、過剰適合の入口にもなる。最初は「単一シグナル + ボラ縮小/拡大時のサイズ制御」がベースライン。

## 理想的ナラティブ / 誤謬リスク

- `BACKTEST_OVERFIT`: シグナルの組み合わせ数が増え、良い過去だけを拾う。
- `PREDICTION_OVERCLAIM`: 複数指標の一致を確信度と誤認する。
- `EXECUTION_GAP`: ボラ急拡大時に想定サイズで逃げられる前提。

## 戦略部品への分解

- `Signal Generator`: MA、breakout、momentumの候補。
- `Position Sizer`: realized volatility、ATR、target volatility。
- `Risk Guard`: 高ボラ停止、連敗停止、日次損失上限。
- `Evaluation Harness`: シグナル数を増やす前後の寄与分解。

## 実験に落とすなら

単一トレンドシグナル、2シグナル、3シグナルを順に比較し、取引回数、期待値、DD、turnover、手数料感応度を見る。

## 採用条件

シグナル追加ではなく、ボラティリティスケーリング単体でDD改善が確認できること。

## 捨て条件

シグナルを増やすほど取引数が減り、偶然の良い期間だけを選んでいる場合。

## 現在性チェック

対象取引所の手数料と最小注文サイズを確認する。
