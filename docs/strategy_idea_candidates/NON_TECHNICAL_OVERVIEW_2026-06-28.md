# 戦略アイディア発想のしくみ 非技術者向けまとめ

## 結論

この repo にある戦略アイディア発想のしくみは、「儲かる戦略を自動で見つける機械」ではありません。

実際の目的は、たくさんの戦略案を同じルールで作り、なぜ残したか、なぜ落としたか、どのデータを見たかを記録し、次の検証へ渡してよい案だけを選ぶことです。

つまり、発想そのものよりも、「思いつきに見える案を、後で検証できる形にそろえること」と「都合のよい結果だけを残さないこと」を重視しています。

## 何をしている仕組みか

この仕組みは、次の順番で戦略案を扱います。

1. 使うデータが信用できるか確認する。
2. 決められた案の型から、検証前の戦略案を複数作る。
3. 似すぎた案、必要な情報が足りない案、危ない条件の案を落とす。
4. 残した案だけでなく、落とした案も一覧に残す。
5. 使ったデータ、期間、条件、落とした理由、選んだ理由を保存する。
6. 人間が読める確認用の資料を作る。
7. 次の検証に回す案だけを、既存の検証手順へ渡す。

大事なのは、最初から「よさそうな案だけ」を見せないことです。落とした案も残すことで、あとから「なぜこの案だけを選んだのか」を確認できます。

## どんな型から案を作るか

現在の基本的な案の型は、次のようなものです。

| 案の型 | かんたんな意味 |
|---|---|
| 流れに乗る案 | 上がる流れ、下がる流れが続くかを見る |
| 値動きの大きさを見る案 | 相場が大きく動き始めたか、静かになったかを見る |
| 取引しやすさを見る案 | 売買のしやすさや価格差を条件に入れる |
| 銘柄同士を比べる案 | 複数の対象を比べ、強いものや弱いものを選ぶ |
| 行き過ぎの戻りを見る案 | 上がりすぎ、下がりすぎの反動を見る |
| 暗号資産の先物向けの案 | 資金調達率、価格差、清算の動きなどを見る |

これらは「勝てる型」という意味ではありません。あくまで、検証前の案を同じ形式で作るための型です。

## この仕組みが必ず残す情報

戦略案を作る時は、次を残す設計になっています。

| 残す情報 | 理由 |
|---|---|
| 使ったデータの場所と内容確認 | 後で同じ条件を再確認できるようにするため |
| データの時刻や利用可能だった時点 | 未来の情報を間違って使わないため |
| 何件の案を作ったか | たくさん試した中の偶然を見落とさないため |
| どの案を残したか | 次に検証する対象を明確にするため |
| どの案を落としたか | 都合の悪い結果を隠さないため |
| 落とした理由 | 同じ失敗を繰り返さないため |
| そのままの成績数字 | 参考値として見るため |
| 探しすぎを考慮した評価状態 | 見かけの良さを過信しないため |
| 人間向けの確認資料 | 技術者以外も判断できるようにするため |

## やってはいけないこと

この仕組みでは、次を禁止しています。

- 成績がよく見える案だけを残す。
- 落とした案を記録しない。
- 未来の情報を使ったかもしれない状態で検証へ進める。
- そのままの成績数字を「発見」や「証明」と呼ぶ。
- AIが出した案を自動採用する。
- 検証前の案を、紙上運用や実取引の許可とみなす。
- 過去データでよく見えただけで、実際に使える戦略だと扱う。

## AIを使う場合の扱い

AIを使う場合でも、repo内から外部AIサービスへ自動送信する設計ではありません。

できることは、人間が任意のAIへ渡すための材料を作ることと、人間が戻したAIの回答を「検証前の案」として取り込むことです。

AIの回答は、よい戦略として採用されるわけではありません。通常の戦略案と同じように、出どころ、内容、足りない情報、検証前であることを記録します。

## 現在できること

現在は、次のことができます。

- 検証前の戦略案一覧を作る。
- 同じ入力なら同じ案が出るようにする。
- 残した案と落とした案を両方保存する。
- 使ったデータの出どころや時刻を保存する。
- 未来の情報を使っていないかを見る最低限の確認をする。
- そのままの成績数字と、過信してはいけない評価状態を分ける。
- 人間向けの確認資料を作る。
- 次の検証へ回す案だけを別形式で書き出す。
- 暗号資産先物向けに、一定の型で大量の検証前アイディアを作る。
- 一部の暗号資産先物の案を、過去データ検証用の形式へつなぐ。
- 外部AIへ渡すための材料をローカルで作り、人間が戻した回答を取り込む。

## まだできないこと

まだ次はできません。

- 実データから自動的に利益の源泉を発見する。
- 全ての戦略案を自動で過去データ検証まで通す。
- 実際の取引コストや板の厚さを完全に測る。
- 紙上運用してよい、実取引してよい、と判断する。
- ウォレット、署名、取引所への注文送信を許可する。
- AIの提案をそのまま採用する。

## どう読めばよいか

この仕組みの出力を読む時は、次の順で見るのが安全です。

1. どのデータを使ったか。
2. 何件の案を作ったか。
3. 何件を落としたか。
4. 何件を次の検証へ回したか。
5. 落とした理由に偏りがないか。
6. 残した理由が、単なる見かけの成績だけになっていないか。
7. 未来の情報を使っていないか。
8. 「検証前の案」であることが守られているか。

この順番で見ると、「良さそうな1件」ではなく、「どういう探索をして、どれだけ落として、なぜ残したか」を確認できます。

## 具体例

たとえば、暗号資産先物の短期戦略を考える場合、この仕組みは次のように動きます。

1. 対象銘柄、価格、出来高、資金調達率などの材料を確認する。
2. 「流れが続くか」「行き過ぎが戻るか」「価格差が大きいか」などの型で案を作る。
3. 必要な情報が足りない案や、同じような案を落とす。
4. 残った案に、最大損失、取引コスト、避ける条件などを付ける。
5. 人間が読むための確認資料を作る。
6. 次の検証へ回す案だけを書き出す。

この時点では、まだ「勝てる戦略」ではありません。次の検証に進めるだけの、整理された検証前の案です。

## この仕組みの価値

一番の価値は、派手な自動発明ではなく、失敗しやすい戦略研究を記録付きで進めることです。

特に役に立つのは次の点です。

- 思いつきを同じ形式で整理できる。
- 落とした理由を残せる。
- たくさん試した事実を隠さない。
- 見かけの好成績を過信しにくい。
- 次の検証へ渡す案だけを分けられる。
- 人間が確認するための材料を作れる。
